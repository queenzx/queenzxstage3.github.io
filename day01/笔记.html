第三阶段 版本控制管理工具 第一天 版本控制管理工具 集中式 CVS SVN 分布式 git git是为了管理Linux内核代码而诞生的 一、命令行 Shell命令 1) pwd,cd,ls - pwd: print working directory 显示当前工作目录/路径 - cd: change directory 改变工作路径(相当于window双击文件夹,返回上一层) 使用:cd 目标文件夹/目录/目标路径 cd . 目标当前文件夹或目录 cd .. 回到上一级 cd ./a 进入到当前文件夹下的a文件夹中(./是指当前文件夹,可以省略不写) cd ../a 进入到上一级文件夹下的a文件夹中 ==绝对路径== cd /d/app 进入到D盘下的app文件夹中 cd ~ 进入到当前登录用户的主目录 - ls: list 显示当前目录中的所有非隐藏文件夹和目录 使用: ls linux下隐藏文件或目录的方式: 以.开头的文件或目录为隐藏 - ls -a 显示所有文件夹和目录,包括隐藏的文件夹和目录 - ls -lt 以列表形式展示详细信息,并按照时间顺序排序(不显示隐藏的文件夹或目录) - ls 路径 显示指定目录中的内容 - ls -R 连同子目录一同显示出来，也就所说该目录下所有文件都会显示出来(显示隐藏文件要加-a参数) - clear 清屏 2)新建文件夹/目录 - mkdir: make directory mkdir 目录路径 mkdir test 当前路径建一个test文件夹 mkdir ./a/b/c 当前目录下a文件夹下b文件夹下新建c文件夹(只要一个文件夹不存在,就会报错) mkdir -p 目录路径 如果父级目录不存在,则先创建父级目录 mkdir -p/app/a/b/c (没有a先创建a,在创建b,在创建c)从父级目录开始创建,然后再创建子级目录 mkdir a b 同时创建多个目录 如果文件夹已经存在,则报错 3) 创建文件 - touch touch 1.text 创建1.txt文件 touch test/1.txt 在当前文件test文件夹下创建1.txt文件 如果touch的文件名已经存在,不会覆盖原来的文件,而是修改原来文件的最后访问时间 touch还可以修改目录的最后访问时间 touch 目录名 4)echo(测试使用) 将数据写入到指定文件,如果文件不存在,就创建 echo 数据 > 文件名 (覆盖模式) echo 数据 >> 文件名 (追加模式->换行显示) 5) vi - vi 文件名 使用命令后,命令行会进入文本编辑界面 此界面分为三种模式: 命令模式: vi命令使用后进入的模式。可以输入a/i进入输入模式,也可以输入:进入底行模式 底行模式: 此模式下可以向文件中输入数据,输入完成后按ESC退出输入模式,进入命令模式 输入模式: :wq 保存并退出 :q 不保存,直接退出 :w 保存,但不退出 :q! 不保存,强制退出 6) cat - catch 获取某个文件的内容 cat 文件名 7) 删除文件夹 - rmdir (remove directory) rmdir 目录名 删除目录(永久删除) rmdir a 只能删除空文件夹(目录)。如果空文件夹(目录)中有其他内容,则删除失败 8) 删除文件 - rm (remove) rm 文件名 rm a 如果要删除非空目录 rm -rf 目录名 rm -rf a rm -rf / 删除根目录(/代表所有) 二、git命令 注: 所有的git命令都是以git开头的 1)配置用户信息 - git config --global user.name "用户名" - git config --global user.email "邮箱" 不配置或配置错误,会报错,tell me who you are 2)初始化本地仓库 - git init 进入到目标文件夹中使用命令git init 初始化成功后,目标文件夹中会自动生成一个隐藏的.git文件夹,同时,git bush命令行中路径后面会多一个(master) - .git文件夹就是本地仓库的版本库 3)查看本地仓库的状态 - git status(只关注工作区和暂存区的状态) 三大区域: 工作区: 仓库所在文件夹,保存文件的地方 暂存区: 版本库中暂存文件 分支所在区: HEAD指向的分支 四种状态:(不分先后顺序) modified: 文件有修改,但是还没有暂存 untracked: 新建的文件,还没有被git管理控制住 staged: 文件被暂存,还没有提交 unmodified: 文件被提交 4)提交 - 提交分两个步骤: 1) 从工作区提交到暂存区 git add 文件名 git add 1.js 2) 从暂存区提交到master分支 git commit -m "提交的信息" git commit -m "1111" - 一步到位操作(没有办法撤销) git commit -a 跳过暂存的方式,直接提交到master分支 撤销修改 git checkout -- 文件名 撤销工作区域的修改(所有的) git checkout -- a 步骤: 1)新建一个文件(cat 文件名) 2)使用git add命令提交到暂存区 3)使用git commit 命令提交到master分支 *每使用一个命令后都用一下git status查看状态* git忽略空文件夹,只关注文件
